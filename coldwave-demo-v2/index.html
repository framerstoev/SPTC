Method

This v2 prototype is experimental. It uses a data-driven resilience-curve method rather than predefined event/recovery windows.

  • Q(t) = speed / baseline speed profile; Q = 1 means baseline-level speed performance.
  • Candidate B: dominant sustained performance drop.
  • 6-hour centered rolling median.
  • Threshold = 0.90 x Q0.
  • k = 3 consecutive observations.
  • Tier 3 metrics are observed operational labels.
  • Tier 1/2 variables are predictor/context layers.
  • Score classes are experimental v0 quantile classes.

Caveats

  • Do not treat this as the stable dashboard score.
  • Delay burden is supplemental user-impact information.
  • Profile volume is not observed event-day volume.
  • Phase detection is sensitive to smoothing, threshold, and noise.
